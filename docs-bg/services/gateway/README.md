# шлюз
> Единната обществена входна точка. Притежава междусекторни проблеми — маршрутизиране, JWT
> проверка, ограничаване на скоростта, поточно предаване — и **нулева бизнес логика**.
**Състояние:** 📋 планирано · **Порт (dev):** 8000 · **База данни:** няма
## Отговорности
- Обратно прокси целия клиентски трафик към вътрешни услуги чрез префикс на пътя
  (`/api/v1/auth/* → identity-service`, `/api/v1/agents/* → agent-service`, …).
- Проверете JWT подписите спрямо JWKS на услугата за идентичност (кеширани); отхвърли неавтентифициран
  заявки, с изключение на публични маршрути (влизане, регистрация, уебкукички, здраве).
- Инжектиране на проверени контекстни заглавки надолу по веригата: `X-User-Id`, `X-Company-Id`, `X-Roles`,
  `X-Request-Id` — и премахва всички предоставени от клиента стойности на тези заглавки.
- Валидирайте заявеното `X-Company-Id` спрямо претенциите за членство на токена.
- Поддържано от Redis ограничаване на скоростта за клас на маршрут (авторизация, чат, по подразбиране, уебкукички).
- Прозрачен SSE/разделен пропуск за потоци от чат (без буфериране).
- Запазване на необработено тяло за маршрути на webhook (Stripe, Brevo) - услугата собственик проверява
  самите подписи.
- CORS, ограничения на размера на тялото, списък с разрешени IP за `/admin` маршрути.
- **Охрана само за четене при представяне на самоличност**: заявки, носещи иск за представяне под чужда самоличност, могат само
  използвайте безопасни методи (GET/HEAD); мутиращите заявки се отхвърлят на ръба. Сесиите
  самите те живеят в услуга за самоличност - шлюзът налага само охраната.
## Изрично извън обхвата
Агрегиране, трансформация на отговор, всяка логика на домейн. Ако клиентът има нужда от агрегат
изглед, притежаващата услуга разкрива крайната точка.
## Таблица за маршрутизиране (първоначална)
| Префикс | Цел |
|---|---|
| `/api/v1/auth`, `/api/v1/users`, `/api/v1/companies` | услуга за идентичност |
| `/api/v1/agents`, `/api/v1/sessions`, `/api/v1/skills`, `/api/v1/memory` | агент-сервиз |
| `/api/v1/knowledge` | знание-услуга |
| `/api/v1/registries`, `/api/v1/clients`, `/api/v1/dashboard` | регистратура-служба |
| `/api/v1/business` (фактури, наличност, разходи) | бизнес-сервиз |
| `/api/v1/documents`, `/api/v1/prices`, `/api/v1/margins`, `/api/v1/kss` | документ-обслужване |
| `/api/v1/billing`, `/api/v1/webhooks/stripe` | таксуване-услуга |
| `/api/v1/integrations` | интеграция-услуга |
| `/api/v1/notifications`, `/api/v1/webhooks/brevo`, `/api/v1/support`, `/api/v1/admin` | платформа-услуга (+ администраторски маршрути за всяка услуга) |

По време на миграцията на удушвача немигрираните префикси сочат към наследения монолит — the
таблицата за маршрутизиране *е* състоянието на миграция (вижте [05 §5](../../05-migration-pros-and-cons.md)).
## Контролен списък за внедряване
- [ ] Приложение FastAPI + httpx-базиран прокси с групиране на връзки и поддръжка за поточно предаване
- [ ] JWKS извличане + кеш + обработка на ротация на ключове
- [ ] Мидълуер за инжектиране/отстраняване на контекстно заглавие
- [ ] Валидиране на искане за членство за `X-Company-Id`
- [ ] Redis ограничител на скоростта (една реализация, конфигурация на кофа за клас на маршрут)
- [ ] SSE преминаването е потвърдено от край до край с агентска услуга
- [ ] Маршрути с необработено тяло на Webhook
- [ ] Защита само за четене на имитация (блокиране на мутиращи методи, когато токенът носи
      иск за представяне под чужда самоличност)
- [ ] Старт на проследяване на OpenTelemetry + `X-Request-Id` генериране
- [ ] Таблица за маршрутизиране като конфигурация (env/yaml), а не код — необходима за удушаващи обръщания
